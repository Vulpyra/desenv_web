# Desenv_Web
Aula01
---Apresentação Inicial
  Apresentado os Objetivos de Aprendizagem com o Plano de Aula, incluindo: Ementa, Conteúdo Programático, Metodologia de ensino, Recursos didáticos, Avaliação e Bibliografia.

---Atividades Práticas Supervisionadas (APS)
  Na APS da disciplina de Web Design, os alunos devem concluir os exercícios apresentandos usando ambos, HTML e CSS. Sua entrega deve ser via relatório, no formato de PDF ou WORD pela plataforma CANVAS.

---Arquitetura de um Site – Definição de escopo, prototipação e UX
  O tema central é a arquitetura e planejamento de projetos web, focando em escopo, usabilidade (UX/UI) e o ciclo de prototipação para validação de ideias.

---Introdução ao Versionamento e Deploy
  Gerenciamento de código e deploy via Git e Github, abrangendo colaboração, automação de processos (CI/CD) e boas práticas para publicação de aplicações.

---Introdução ao JavaScript (Vanilla JS)
  Este tópico foca nos fundamentos do JavaScript (Vanilla JS), destacando sua importância como linguagem nativa para controle da lógica, DOM e APIs no front-end.

---Servidores e opções de hospedagem / CMS
  Aborda o gerenciamento de conteúdo via CMS, focando na criação e hospedagem de sites de forma visual e modular, com baixa necessidade de código.
  +Vantagens: Ágeis, baratos e fáceis.
  -Limitações: Restrição técnica, desempenho inferior e dependência da plataforma.

---Website Builders
  Resume o uso de Website Builders, destacando a agilidade e facilidade de uso contra limitações de performance, customização e migração.
  +Vantagens: Agilidade na entrega, facilidade de uso sem código e baixo custo com hospedagem inclusa.
  -Limitações: Restrições em customizações específicas, performance inferior e dificuldade de migração de dados.

---Introdução a Frameworks Front-end
  O desenvolvimento moderno com Frameworks (React, Vue e Angular), focando em produtividade e organização do código.
  =​Arquitetura: Uso de componentização para reutilização de código e melhor estruturação do projeto.
  =​Funcionalidades: Gerenciamento de estado, rotas dinâmicas e integração eficiente com APIs externas.
  
---Introdução a Frameworks Back-end
  Foca no desenvolvimento de APIs e lógica de servidor, utilizando frameworks para estruturar o back-end de forma segura e escalável.
  ​=Arquitetura: Aplicação de padrões como MVC e criação de APIs RESTful para organizar a comunicação de dados.
  =​Segurança e Dados: Foco em integração com bancos de dados, autenticação de usuários e proteção da aplicação.
  
---Introdução a Banco de Dados NoSQL
  Armazenamento não-relacional com NoSQL, focado em lidar com grandes volumes de dados de forma ágil e distribuída.
  =​Flexibilidade: Uso de esquemas dinâmicos que permitem armazenar diferentes tipos de dados sem tabelas fixas.
  =​Performance: Alta escalabilidade horizontal, facilitando o crescimento do sistema em múltiplos servidores.

---Critérios de Avaliação
  =AC1(15%): Prova teórica de múltipla escolha.
  =AC2(30%): Entrega do projeto prático, apresentação oral e benchmarking.
  =AG(10%): Avaliação online institucional.
  =AF(45%): Entrega da documentação final robusta do projeto (formato de um "mini" TCC).
